# データ設計

この資料は、保存データと将来の保存先変更に関する内部確認用です。  
通常作業では `操作手順.md` と `確認項目.md` を使います。

## 目的

このPWAは、iPad / iPhone Safariで工事写真台帳を作成し、PDFで顧客へ提出するための構成です。  
現在の保存方式はJSONファイル出力です。将来、SharePoint Listsへ切り替えやすいように、保存処理は `js/config.js` の `saveInputData(data)` に分離しています。

## ファイル構成

|場所|役割|
|---|---|
|`index.html`|画面の起点|
|`css/style.css`|画面・印刷レイアウト|
|`js/config.js`|設定・保存処理|
|`js/app.js`|入力・写真・PDF処理|
|`manifest.webmanifest`|ホーム画面追加用設定|
|`service-worker.js`|同梱ファイルのキャッシュ|
|`icons/`|ホーム画面用アイコン|
|`docs/`|操作説明・確認資料|

## JSON保存データ

保存ボタンで出力されるJSONの最上位構造です。

```json
{
  "cover": {},
  "compression": "middle",
  "entries": [],
  "savedAt": "2026-06-10T00:00:00.000Z"
}
```

## cover

表紙に表示する情報です。

|項目|型|内容|
|---|---|---|
|`siteName`|文字列|現場名|
|`workName`|文字列|工事名|
|`startDate`|文字列|着工|
|`finishDate`|文字列|竣工|
|`contractorName`|文字列|施工者|
|`showContractor`|文字列|表示有無|

`showContractor` は `yes` または `no` です。

## compression

写真圧縮設定です。

|値|内容|
|---|---|
|`large`|強く圧縮|
|`middle`|標準|
|`small`|画質優先|

実際のピクセル上限とJPEG品質は `js/config.js` の `PRESETS` で管理しています。

## entries

写真枠の配列です。1枠ごとに1件です。

|項目|型|内容|
|---|---|---|
|`no`|文字列|写真番号|
|`work_type`|文字列|工種|
|`notes`|配列|自由入力5行|
|`image`|文字列|画像データ|
|`fit`|文字列|表示方法|

`fit` は `cover` または `contain` です。

## 下書き保存データ

入力中の内容は端末内にも下書き保存します。写真枚数が多く容量を超える場合は、写真を除いた文字情報のみを下書き保存し、`liteDraft` が `true` になります。

```json
{
  "cover": {},
  "compression": "middle",
  "entries": [],
  "savedAt": "2026-06-10T00:00:00.000Z",
  "liteDraft": true
}
```

## 保存処理の分離

現在の保存処理は以下です。

```text
app.js の exportJson()
  ↓
config.js の saveInputData(data)
  ↓
JSONファイルとしてダウンロード
```

SharePoint Listsへ切り替える場合は、`saveInputData(data)` の中身だけを変更します。画面側の入力処理、写真処理、PDF出力処理は基本的に触らない想定です。

## SharePoint Lists変更時の候補分割

|保存先候補|保存内容|
|---|---|
|写真帳リスト|cover情報|
|写真明細リスト|entries情報|
|ドキュメントライブラリ|写真ファイル|
|PDF出力履歴|提出PDFの記録|

現時点では外部送信は行っていません。

## PDF出力設計

PDFはブラウザの印刷機能を利用します。`app.js` の `buildPrintPages()` で印刷専用DOMを生成し、`printPdf()` で印刷画面を開きます。写真は1ページ3枚構成です。表紙は1ページ固定です。
